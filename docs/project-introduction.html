农田智能监测与巡田决策平台

面向精准农业的农田监测与巡田决策系统，围绕地块管理、遥感影像处理、异常检测、巡田工单调度、路径规划和 AI 助手构建完整业务闭环。

Spring BootMySQLLeafletCesiumPython FlaskFunction CallingSSE

项目背景

传统农田管理依赖人工巡检和经验判断，地块信息、遥感影像、异常检测结果与巡田任务之间缺少统一流转。该项目以地块为核心对象，将地图圈地、影像处理、异常检测、工单生成和路线规划串联起来，提升农田监测和巡田任务生成效率。

核心功能

地块管理支持地块新增、编辑、查询、地图定位、边界坐标保存和地块属性维护。
二维与三维可视化基于 Leaflet 和 Cesium 展示地块、影像、空间位置和三维场景。
遥感影像处理通过 Python Flask 服务按地块边界裁剪 GeoTIFF 影像，避免整幅影像传输。
异常检测与工单对遥感结果进行异常识别，并生成巡田任务单和巡田报告。
巡田路径规划根据异常工单和地块位置生成巡田路线，辅助任务执行。
AI 助手集成 Function Calling 与 SSE，支持地块问答、异常分析和巡田辅助决策。

AI 能力

系统 AI 模块采用大模型对话 + 工具调用的方式，不只是普通聊天。管理员可以在地块管理页面使用自然语言查询地块，例如“帮我查一下沽源的地块”“有哪些种植玉米的地块”“面积最大的地块是哪块”。

  • query_plots：按地块名称、地区、地址、作物、土壤类型、灌溉方式和面积范围查询地块。
  • get_anomaly_records：查询巡田异常工单列表和详情。
  • trigger_anomaly_detection：触发异常检测任务。
  • generate_patrol_route：基于异常工单生成巡田路径。
  • SSE 流式输出：前端展示 AI 回复、Agent 执行过程和工具调用结果。

技术架构

项目采用“Java 业务层 + Python 计算层 + 可视化层”的分层架构。

  • Java 业务层：Spring Boot 负责用户管理、地块管理、巡田管理、权限校验、AI 接口和数据库访问。
  • Python 计算层：Flask 提供遥感影像裁剪、异常检测、路径规划等计算接口。
  • 前端可视化层：HTML、JavaScript、Leaflet、Cesium 实现地图交互、二维圈地和三维展示。
  • 数据层：MySQL 存储用户、地块、工单、巡田记录和业务数据。

项目亮点

  • 围绕农田业务形成“地块圈定 -> 影像裁剪 -> 异常识别 -> 工单生成 -> 路径规划”的闭环。
  • 通过 GeoTIFF 按需裁剪减少影像传输体积，提升地图加载和交互体验。
  • 通过异步巡田任务处理机制获取异常检测结果，降低长耗时任务对主业务流程的阻塞。
  • 基于 Function Calling 构建 AI Agent，使大模型能够调用业务工具查询地块、分析异常并生成巡田建议。
  • 管理员地块 AI 问答已接入地块列表和地图管理页面，能够直接服务实际管理场景。
