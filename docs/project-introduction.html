农田智能监测与巡田决策平台完整项目介绍

该项目面向精准农业场景，围绕“地块圈定、遥感影像处理、异常检测、巡田工单调度、路径规划和 AI 助手”构建一套农田监测与巡田决策系统。

Spring BootMySQLLeafletCesiumPython FlaskFunction CallingSSEAI Agent

1. 项目背景

农田管理中，地块位置、作物信息、遥感影像、异常检测结果和巡田任务通常分散在不同系统或人工流程中。管理人员即使发现异常，也需要再手动查找地块、确认联系人、安排巡田路线，效率较低。

本项目以地块为核心对象，将地图空间数据、遥感影像处理、异常检测和巡田调度串联起来，让系统不仅能“发现问题”，还可以继续生成工单、规划路线，并通过 AI 助手辅助查询和决策。

2. 业务流程

地块圈定

用户或管理员在地图上绘制地块边界，保存坐标、面积、作物、土壤、灌溉、联系人和地址等信息。

影像裁剪

系统将地块边界提交给 Python Flask 服务，对 GeoTIFF 遥感影像进行按需裁剪，减少整幅影像传输和渲染压力。

异常检测

对裁剪后的影像或指定区域执行异常检测，识别可能存在长势异常、缺水、病虫害等风险的区域。

工单生成

将异常检测结果转化为巡田任务单，记录异常类型、地块名称、建议处理方式和相关图片。

路径规划

根据异常工单和空间位置生成巡田路线，让巡田任务具备可执行路径。

AI 辅助

管理员可通过 AI 对话查询地块、分析异常、触发检测和生成巡田路线，降低操作门槛。

3. 系统架构

项目采用“Java 业务层 + Python 计算层 + 前端可视化层 + MySQL 数据层”的分层结构，实现业务管理和算法计算解耦。

Java 业务层Spring Boot 负责用户管理、地块管理、巡田管理、AI 接口、权限校验和数据库访问。
Python 计算层Flask 提供遥感影像裁剪、异常检测和路径规划等计算接口。
前端可视化层HTML、JavaScript、Leaflet、Cesium 实现地图圈地、二维展示、三维可视化和 AI 聊天窗口。
数据存储层MySQL 存储用户、管理员地块、用户地块、巡田工单、异常记录和业务配置。

4. 核心模块

模块实现内容项目价值
地块管理地块增删改查、边界坐标、面积、作物、土壤、灌溉、联系人和地址维护。建立统一地块数据底座。

5. AI Agent 能力

AI 模块采用大模型对话 + Function Calling 的方式，将业务能力封装为工具，让大模型根据用户问题选择合适工具，而不是只生成通用回答。

query_plots

地块查询工具

按地块名称、地区、地址、作物、土壤、灌溉方式、土地类型和面积范围查询地块。

get_anomaly_records

异常工单查询

查询巡田异常工单列表和详情，为异常解释和路径规划提供上下文。

trigger_anomaly_detection

触发异常检测

根据地块、日期范围和区域参数触发异常检测任务。

generate_patrol_route

生成巡田路线

基于异常工单生成巡田路径，形成执行方案。

explain_anomaly

异常解释

结合异常类别和地块信息输出可能原因、影响和巡田建议。

SSE

流式过程展示

前端实时展示 AI 回复和 Agent 工具调用过程，增强交互反馈。

6. 管理员地块 AI 问答

管理员地块问答是项目后续补充的重要能力，已经接入地块列表和地图管理页面。它的目标是让管理员不用手动组合筛选条件，也能通过自然语言查询地块数据。

可支持的问题示例

  • “帮我查一下沽源的地块。”
  • “有哪些种植玉米的地块？”
  • “面积最大的地块是哪一块？”
  • “系统里一共有多少亩地？”
  • “砂质土并且渠灌的地块有哪些？”

实现方式

后端通过地块查询工具统一封装数据库查询能力，AI 根据用户输入抽取地区、作物、面积、土壤、灌溉等条件，再调用工具获取真实数据。工具返回后，AI 再组织成自然语言答案。如果查询结果为空，系统明确说明数据库中没有查到，而不是编造结果。

7. 巡田任务闭环

巡田模块的重点不是单独展示异常，而是把异常检测结果继续向下游任务推进。

异常识别

从遥感影像或地块数据中识别风险区域，形成异常记录。

工单沉淀

将异常记录转化为巡田任务单，包含地块、联系人、异常类型和处理建议。

路线输出

基于异常地块位置生成巡田路线，辅助巡田人员安排巡检顺序。

8. 技术难点与解决方案

难点问题表现解决方案
大影像加载慢GeoTIFF 体积大，整幅影像传输和渲染成本高。使用 Python Flask 根据地块边界动态裁剪，前端按需加载裁剪结果。
长耗时检测任务异常检测无法在普通请求中快速完成。设计异步任务处理方式，通过轮询获取检测结果，避免阻塞主流程。
业务数据和 AI 结合大模型如果直接回答容易脱离真实数据库。通过 Function Calling 暴露业务工具，AI 必须先查真实数据再组织答案。
地图空间交互地块边界、影像、定位和三维展示需要统一表达。Leaflet 负责二维绘制和地图交互，Cesium 负责三维场景展示。
AI 输出体验普通接口一次性返回，用户等待感明显。采用 SSE 流式输出，展示 AI 回复和 Agent 执行过程。

10. 项目总结

该项目不是单纯的后台管理系统，而是把农田空间数据、遥感影像处理、异常检测、巡田任务和 AI 助手组合到一个农业业务闭环中。它体现了 Java 后端业务开发、Python 算法服务集成、地图可视化和大模型工具调用的综合实践。

在简历和面试中，可以重点强调三点：第一，系统采用 Java 业务层与 Python 计算层解耦；第二，遥感异常检测结果可以继续生成巡田工单和路径；第三，AI Agent 通过 Function Calling 调用真实业务工具，具备地块问答和巡田辅助决策能力。
