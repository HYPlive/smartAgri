ABOUT PROJECT

面向精准农业的全流程监测与巡田决策系统

项目以地块为核心数据对象，结合二维地图编辑、三维场景展示、遥感影像裁剪、农田异常识别、巡田工单和 AI Agent 助手，实现“地块圈定 -> 影像裁剪 -> 异常识别 -> 工单生成 -> 路径规划 -> AI 辅助决策”的闭环。

FEATURES

核心功能模块

覆盖农业平台后台管理、地图交互、遥感数据处理和巡田任务输出。

01

地块管理与 AI 圈地

支持地块边界绘制、信息维护、地图定位和 AI 辅助圈地，统一管理地块空间数据。

02

遥感异常检测

结合遥感影像与异常结果面板，辅助识别作物长势、区域异常和巡检重点。

03

巡田路径规划

从异常工单出发生成巡田路线，减少人工筛选和重复规划成本。

04

巡田工单调度

异常检测结果可以沉淀为巡田任务单，形成后续处理依据。

05

巡田报告输出

对异常区域、巡田建议和路线结果进行结构化展示，便于汇报和执行。

06

AI Agent 助手

通过 Function Calling 调用业务工具，支持异常分析、地块查询和巡田辅助决策。

AI CAPABILITY

管理员地块问答与巡田 Agent

AI 模块不是单纯聊天，而是将自然语言输入转换为业务工具调用，再把数据库和算法服务结果组织成可读答案。

TOOL 01

query_plots

支持按地块名称、作物、地区、地址、土壤类型、灌溉方式和面积条件查询管理员地块数据。

TOOL 02

get_anomaly_records

查询巡田异常工单列表和详情，为后续异常解释和路径规划提供上下文。

TOOL 03

trigger_anomaly_detection

根据地块、时间范围和区域参数触发异常检测任务，适配异步处理流程。

TOOL 04

generate_patrol_route

基于异常工单生成巡田路线，让检测结果能转化为可执行任务。

STREAM

SSE 流式输出

前端展示 AI 回复、工具调用过程和结果摘要，提升交互反馈速度。

ADMIN

地块管理 AI 聊天

在地块列表和地图管理页面接入聊天入口，支持管理员直接用自然语言查询地块。

ARCHITECTURE

Java 业务层 + Python 计算层 + 可视化层

平台将业务管理、空间计算和前端地图展示拆开：Spring Boot 负责权限、地块、工单和 AI 编排；Python Flask 承担遥感影像裁剪与异常检测；Leaflet 与 Cesium 完成二维编辑和三维展示。

业务服务层Spring Boot、JWT、MySQL、地块管理、巡田管理、AI Controller
算法计算层Python Flask、GeoTIFF 裁剪、异常检测、路径规划接口
可视化交互层HTML、JavaScript、Leaflet、Cesium、地图圈地与空间展示
AI 编排层Function Calling、SSE、Agent 工具调用、地块问答和巡田辅助
